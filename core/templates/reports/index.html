Inventory control tools
Inventory ↔ GL reconciliation

Flags any tenant whose inventory subledger drifts from the GL control account (1000). Run on a schedule:

python manage.py check_inventory_gl
python manage.py check_inventory_gl --tolerance 0.50

Tenant-level figures are authoritative. Per-location is subledger-only — inter-site transfers post no GL entry, so site-level isn't expected to balance.

Cycle-count valuation catch-up

Revalues historical cycle-count movements that used product-average to lot/serial-specific cost, posting auditable revaluation entries. Run dry-run first — it only reports until you pass --apply; it never rewrites the original movements.

# 1) Review (no changes):
python manage.py reconcile_cycle_count_valuation
# 2) Apply, treating periods up to a close date as locked
#    (those corrections post into the current open period,
#     referencing the original movement):
python manage.py reconcile_cycle_count_valuation --apply --lock-date 2026-03-31

Idempotent — re-running never double-corrects. Lots with no cost layer fall back to the product average (documented).
